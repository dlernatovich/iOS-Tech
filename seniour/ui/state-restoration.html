Як підходити до state restoration?

Коротка відповідь

State restoration має відновлювати важливий user context після app kill, memory pressure або deep link. Не обов'язково зберігати весь UI tree; краще зберігати domain state, route і мінімальні параметри.

Важливо врахувати privacy, stale data, auth state, schema migrations і конфлікти між restoration та incoming deep link/push notification.
