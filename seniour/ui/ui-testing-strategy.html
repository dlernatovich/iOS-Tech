Якою має бути UI testing strategy?

Коротка відповідь

UI tests мають покривати критичні user journeys, а не дублювати всі unit tests. Для стабільності потрібні accessibility identifiers, контрольовані test doubles, predictable launch arguments, seed data і мінімум залежності від реального backend.

Senior-рівень: балансувати test pyramid, snapshot tests, integration tests, flakiness budget і CI runtime.
