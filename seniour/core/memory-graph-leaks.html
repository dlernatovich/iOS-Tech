Як шукати memory leaks в iOS app?

Коротка відповідь

Починають із Memory Graph Debugger, Instruments Leaks/Allocations, deinit logs і сценаріїв відкриття-закриття екранів. Типові причини: retain cycles у closures, delegates без weak, timers, NotificationCenter, Combine subscriptions, async tasks.

service.load { [weak self] result in
    guard let self else { return }
    self.render(result)
}

Senior-рівень: не тільки знайти leak, а й пояснити ownership model і додати regression-перевірку там, де це можливо.
