Storyboard, XIB чи UI in code?

Коротка відповідь

Storyboard зручний для невеликих flow і швидкої візуальної роботи, але може створювати merge conflicts і приховані runtime-зв'язки. XIB добре підходить для окремих reusable views. UI in code краще контролюється, рев'юється і тестується.

Сильна відповідь: вибір залежить від команди, складності навігації, дизайн-системи і вимог до maintainability.
