Що таке ARC, strong, weak та unowned?

Коротка відповідь

ARC автоматично рахує strong-посилання на об'єкти класів і звільняє пам'ять, коли посилань не лишається. weak не утримує об'єкт і стає nil. unowned також не утримує, але не optional і впаде, якщо об'єкт уже звільнений.

Типовий кейс: delegate майже завжди weak, щоб уникнути retain cycle.

protocol ScreenDelegate: AnyObject {
    func didTapClose()
}

final class ScreenViewModel {
    weak var delegate: ScreenDelegate?
}
