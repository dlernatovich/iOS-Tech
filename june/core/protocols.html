Для чого потрібні protocols і protocol extensions?

Коротка відповідь

Protocol описує контракт: які властивості або методи має мати тип. Extension може додати дефолтну реалізацію, що допомагає зменшити дублювання.

Це основа delegation, dependency inversion, тестування через mocks і багатьох стандартних Swift API.

protocol Trackable {
    func track(_ event: String)
}

extension Trackable {
    func trackScreen(_ name: String) {
        track("screen_opened:\(name)")
    }
}
