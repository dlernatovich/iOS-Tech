Що таке Dependency Injection і навіщо воно в iOS?

Коротка відповідь

Dependency Injection означає, що об'єкт отримує залежності ззовні, а не створює їх сам. Це зменшує coupling, спрощує тестування і дозволяє підміняти реалізації.

protocol UserService {
    func loadUser() async throws -> User
}

final class ProfileViewModel {
    private let service: UserService

    init(service: UserService) {
        self.service = service
    }
}
