Middle / UI

Як працює data flow у SwiftUI?

Коротка відповідь

Основні інструменти: @State для локального стану, @Binding для двостороннього зв'язку, @StateObject для object, який створює і яким володіє view, @ObservedObject для object, який передали ззовні, і @EnvironmentObject для shared state.

Головна ідея: UI перераховується при зміні state, тому side effects, бізнес-логіку і роботу з сервісами варто ізолювати у view models/services.

MVVM у SwiftUI: ObservableObject + ObservedObject + View

У MVVM для SwiftUI View відповідає за відображення, ViewModel тримає стан, виконує дії користувача і викликає сервіси. Якщо view отримує view model ззовні, використовуй @ObservedObject. Якщо view сама створює view model і має володіти її lifetime, використовуй @StateObject. 

import SwiftUI

struct User: Equatable {
    let name: String
    let email: String
}

protocol UserLoading {
    func loadUser() async throws -> User
}
@MainActor
final class ProfileViewModel: ObservableObject {
    enum State: Equatable {
        case idle
        case loading
        case loaded(User)
        case failed(String)
    }

    @Published private(set) var state: State = .idle

    private let loader: UserLoading

    init(loader: UserLoading) {
        self.loader = loader
    }

    func load() async {
        state = .loading

        do {
            let user = try await loader.loadUser()
            state = .loaded(user)
        } catch {
            state = .failed("Не вдалося завантажити профіль")
        }
    }
}
struct ProfileView: View {
    @ObservedObject var viewModel: ProfileViewModel

    var body: some View {
        content
            .task {
                await viewModel.load()
            }
    }

    @ViewBuilder
    private var content: some View {
        switch viewModel.state {
        case .idle, .loading:
            ProgressView("Завантаження...")

        case .loaded(let user):
            VStack(alignment: .leading, spacing: 8) {
                Text(user.name)
                    .font(.headline)
                Text(user.email)
                    .foregroundStyle(.secondary)
            }

        case .failed(let message):
            VStack(spacing: 12) {
                Text(message)
                Button("Спробувати ще раз") {
                    Task { await viewModel.load() }
                }
            }
        }
    }
}
struct ProfileContainerView: View {
    @StateObject private var viewModel: ProfileViewModel

    init(loader: UserLoading) {
        _viewModel = StateObject(
            wrappedValue: ProfileViewModel(loader: loader)
        )
    }

    var body: some View {
        ProfileView(viewModel: viewModel)
    }
}

Що важливо пояснити на співбесіді

  • ObservableObject повідомляє SwiftUI про зміни через @Published.
  • @ObservedObject не володіє object lifecycle, а лише спостерігає за переданою view model.
  • @StateObject потрібен там, де view створює object і має зберегти його між перерахунками body.
  • ViewModel краще робити @MainActor, якщо вона оновлює UI state.
  • Сервіс краще вводити через protocol, щоб ViewModel було легко тестувати.
